2.2 post  /survivors
Summary Add survivors to the database and register theirs inventories
Description A survivor must have a name, age, gender and last location (latitude, longitude).
Operation Id addSurvivor
Produces application/json
Consumes application/json
Parameters
Name In Description Required Type Format Collection Format Default Min Max
survivor body Survivor to add
Name Type Required
name string Yes
age integer Yes
gender string Yes
last_location string Yes
resources_attributes array Yes
Yes
Body Example 
{
  "name": "Max Chow",
  "age": 28,
  "gender": "M",
  "last_location": "36.800488,-89.121094",
  "resources_attributes": [
    {
      "name": "Water"
    }
  ]
}
Responses
code description
201 Survivor created
422 invalid input, object invalid
Response Example 
{
  "id": 2,
  "name": "Max Chow",
  "age": 28,
  "last_location": "36.800488,-89.121094",
  "gender": "M",
  "infected": false,
  "resources": [
    {
      "id": 2,
      "name": "Water",
      "points": 4,
      "survivor_id": 2,
      "created_at": "2017-11-01T11:27:19.516Z",
      "updated_at": "2017-11-01T11:27:19.516Z"
    }
  ]
}

2.5 post  /flags
Summary flag a survivor as infected
Description a survivor flags another as infected and after 3 flags the victim will be marked as infected
Operation Id flagSurvivor
Produces application/json
Consumes application/json
Parameters
Name In Description Required Type Format Collection Format Default Min Max
flag body a survivor's flagging another as infected
Name Type Required
reporter_id integer Yes
infected_id integer Yes
Yes
Body Example 
{
  "reporter_id": 1,
  "infected_id": 2
}
Responses
code description
204 the victim was flagged as infected
422 the arguments are invalid
Response Example 
{
  "reporter_id": 1,
  "infected_id": 2
}
